游戏名字：诗词名句辞典
游戏类型：小游戏 
开发环境：Vscode 
开发语言：python
设计尺寸：420*800
操作方式：触屏操作
玩法说明：在游戏主界面，呈现给玩家一组字，文字是由配置表配置的由正确文字+干扰文字组成，玩家选择其中的部分字，组成一组诗句，如果正确，那么游戏以满分通关。选中正确的字会显示在底部的框中，错误的字选中之后，会扣除1次选择机会，当机会次数为0，那么游戏结束，需要重新开始。选错之后，扣除10分。

界面：游戏有加载界面、主界面、设置界面、结算界面等5个界面组成，其中设置界面中会有朝代列表界面，朝代列表界面点击后进入到关卡详情界面。

UI交互逻辑
1、加载界面
加载界面确保进度条结束之后进入游戏主界面。

2、主界面
点击设置按钮，打开设置界面；
点击提示按钮的逻辑：点击之后显示配置表中的内容，第1次点击“提示”，在提示1位置显示csv配置表中tips1的内容，第2次点击，在提示2位置显示csv配置表中tips2的内容。显示之后，固定显示；当关卡结算，会清空提示的信息；
撤销：当错误选择＞=1的时候，按钮替换成同目录下的cexiao.png,且可以被点击，点击之后，清除掉文字选择框上面的“错误效果”； 当错误选择＜1的时候，按钮替换成cexiao_1.png;
刷新：点击后，重新读取当前关卡的配置信息，重新生成一批all_text在文字选择区域。
错误选择是指，玩家选中了非csv配置表wenzitext的文字；
正确选择是指，玩家选中了csv中配置表中wenzitext的文字；
当错误选择的时候，会在错误选择的文字框上面覆盖一个图层，用图片目录下的cuo.png；
当正确选择的时候，会在正确选择的文字框上面覆盖一个图层，用图片目录下的dui.png；
同时，正确选择的文字会根据它在wenzitext的顺序，出现在底部的文字显示框中，因此，wenzitext中的文字的顺序是固定的，且会被文字显示框区域调用。

游戏逻辑
文字选择区域的信息会根据csv关卡配置的信息刷新文字给玩家选择，信息包括：（1）id，数据类型int，是关卡的id，关卡的id是唯一的，决定了所在行也就是2~6的内容；（2）wenzitext，数据类型string，字符串的长度决定了正确的点击次数，字符串中字符的顺序决定了它在文字显示区域的排序以及文字显示区域的显示框的数量；（3）clicktimes，数据类型int，意思是可点击的次数；；（4）errortext，数据类型int，意思是用来干扰的文字的数量，这里并不直接读取具体的文字，而是确定干扰性文字的数量，再从从当前目录下的“3500word.txt“中抽取，抽取时，会跟wenzitext的文字去重，确定没有相同的文字之后再抽取，抽取到的文字与wenzitext中的文字一起组合成all_text，all_text的文字数量多少就有多少个文字选择框；（5）tps1，数据类型string，第一个提示的意思；（6）tps2，数据类型string，第二个提示的意思；

点击区域的内容显示规则：
读取3500word.txt的文字，并独立成一个个独立的字符串，定义成words；
从csv文件中读取wenzitext，把wenzitext中的文字拆分成一个个字符串，定义为correct_texts；
从errortext获取干扰性文字的数量，确定干扰性文字的数量后，再从words中抽取对应数量的字符串，抽取时需要排除掉correct_texts的内容，字符串一起组合成all_text，all_text的字符串数量决定了有多少个文字选择框，每个文字选择框中有一个字符串；

关卡的读取规则：初始默认从configs目录读取xianqin表，且读取id=1的这一行的信息， 如果有记录，那么从玩家上一次记录中读取关卡信息；

计分规则

游戏逻辑
当选择次数＜=0，或者已经选择完所有的righttext，进行结算，在主界面之上弹出结算界面，根据玩家的分数，判断是否进行游戏信息的存储。
分数的计算：选择文字的时候，执行以下规则：
如果righttext文字数= 4个，那么每个文字25分；
如果righttext文字数=5个，那么每个文字20分；
如果righttext文字数=6个，那么前5个文字每个15分，第6个25分；
如果righttext文字数=7个，那么前6个文字每个15分，第6个10分；
如果righttext的文字=8个，那么前7个文字每个10分，第8个20分；
错误操作一次，分数为-10分。

信息存储
信息存储分成3部分功能，第一部分是记录玩家在每一关的分数；第二部分是记录玩家当前解锁的关卡；还有一个用来记录玩家当前所处的关卡。
如果分数＞=60，玩家点击“继续游戏“，那么记录游戏的分数到“game_save.json”的文件中，进度记录包括：当前的朝代以及关卡的id、分数；如xianqin,id=100，分数=70.如果当前的关卡是第一次解锁的关卡，那么会记录解锁的信息到unlock_save.json中，记录的信息包括当前的朝代以及关卡的id，下一关就表示是未解锁的关卡。
当运行游戏或者打开游戏的时候，读取当前配置表的下一个id的配置，如果当前配置表已经结束，那么按照（1）xianqin，先秦；（2）han，两汉；（3）sanguo，三国~南北朝；（4）tang，隋唐；（5）song，五代两宋；（6）mingqing，元明清；（7）jindai，近当代；（8）other，其它 的顺序读取配置表，不管玩家是否已经解锁该关卡。
当游戏重新如果读取的配置表时，如果是最后一个配置表的且没有未完成的关卡，那么显示文字“已完成全部学习”
